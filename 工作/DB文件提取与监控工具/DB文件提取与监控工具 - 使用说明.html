📁 DB文件提取与监控工具

用户使用说明文档

🎯 核心功能

  • 手动提取：从 .db 二进制文件中提取结构化数据（到 .txt）和图片
  • 后台监控：自动监控文件夹，将新生成的 .txt 文件复制到指定位置

📤 功能一：手动提取 .db 文件

▼

当您需要将一个 .db 文件转换为可读的 .txt 和图片时，请使用此功能。

步骤 1：启动程序

打开 .exe 文件。

步骤 2：选择输入文件

在"手动 .db 文件提取"区域，点击 输入 .db 文件 旁边的 ... 按钮，选择您要转换的 .db 文件。

步骤 3：选择文本输出

点击 输出 .txt 文本路径 旁边的 ... 按钮，指定保存位置和文件名（例如 output.txt）。

步骤 4：选择图片输出

点击 输出图片目录 旁边的 ... 按钮，选择一个文件夹保存提取的图片。

步骤 5：开始转换

点击 手动转换选中 .db 文件 按钮。

步骤 6：检查结果

查看底部的"日志"框。如果一切顺利，您会看到：

  • [提取] 缺陷图片: ...
  • 手动数据导出完成！

您的文件和图片现在已经生成在指定路径。

🔄 功能二：后台自动转移 .txt 文件

▼

本工具的核心功能，监视文件夹并自动复制新的 .txt 文件。

步骤 1：设置日志（重要！）

程序默认启用日志记录，必须指定日志路径才能启动后台服务。

  • 前往 日志设置 区域
  • 确保 "启用日志，保存到文件：" 已被勾选（默认设置）
  • 点击 ... 按钮，选择位置和文件名（例如 C:\MyLogs\monitor.log）

步骤 2：设置监控路径

前往 后台 .txt 文件自动转移 区域：

  • 源文件夹：点击 ... 选择要监视的文件夹（本地路径如 C:\Data 或共享盘路径如 \\Share\Data_AI\...）
  • 目标文件夹：点击 ... 选择文件复制目标位置（例如本地桌面）
  • 刷新间隔(秒)：输入检查频率，默认 60 秒（1分钟）

步骤 3：测试路径（推荐）

在正式启动前测试路径配置：

  • 点击 转移测试 按钮
  • 程序将扫描源文件夹中所有 .txt 文件并尝试复制
  • 检查日志框，查看 [监控] 成功复制: ... 或 [监控] 跳过: ... 消息

步骤 4：启动后台服务

  • 点击 启用后台持续运行 按钮
  • 按钮文本变为 "关闭后台持续运行"
  • 程序执行一次全盘扫描，然后按设定间隔检查新文件
  • 日志框显示详细的启动参数和检查状态

步骤 5：最小化到托盘

后台服务运行时（按钮显示"关闭..."），点击窗口右上角 X：

  • 程序不会退出，而是最小化到系统托盘（屏幕右下角，时钟旁边）
  • 在后台持续运行
  • 双击托盘图标可恢复窗口查看日志

⚠️ 注意事项（请务必阅读！）

▼

🚪 如何彻底退出程序？

  • 后台未运行（按钮显示"启用..."）：直接点击 X 即可退出
  • 后台正在运行（已最小化到托盘）：在系统托盘右键点击程序图标，选择 "退出程序"

⚙️ 程序默认状态

  • 为了安全，每次启动时后台监控默认为 关闭 状态
  • 需要手动点击"启用后台持续运行"来启动
  • "启用日志"默认是开启的

📝 日志路径是必需的

  • 如果"启用日志"被勾选但未指定有效的 .log 文件路径
  • 程序将拒绝启动后台运行，并弹窗提示

🔐 权限问题

  • 确保 Windows 账户对 源文件夹（尤其是公共盘）有 读取权限
  • 对 目标文件夹 和 日志文件路径 有 写入权限
  • 否则日志中会显示"访问被拒绝"错误

🔍 "转移测试" vs "启用"

  • 转移测试：扫描所有文件（从1970年开始），不更新"上次检查时间"，可反复点击测试
  • 启用：第一次点击扫描所有文件，然后更新"上次检查时间"为现在，此后只检查新文件

💡 如有疑问，请查阅日志文件获取详细信息
